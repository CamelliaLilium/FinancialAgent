# FinBen 多智能体 Base 测试：中文错误分析报告

## 1. 基本结果概览

- 测试集规模：496
- 正确数：247
- 准确率：0.4980
- 平均单样本耗时：34.308 秒

结论：在当前任务（短文本三分类）上，多智能体工作流表现为“准确率中等、时延较高、流程开销明显”。

## 2. 误差与过程信号统计

### 2.1 主要错误标签分布

- `reasoning_or_prompt_misalignment`: 249  
  含义：模型语义判断与标注口径不一致，是当前准确率下降的主要直接原因。
- `tool_execution_error`: 35  
  含义：工具调用或执行异常，属于流程层不稳定因素。
- `repeated_tool_loop`: 11  
  含义：在个别样本上出现循环行为，拉长时延并增加随机错误概率。
- `likely_unnecessary_web_search`: 2  
  含义：少量样本存在不必要检索调用。

### 2.2 工具调用结构（过程开销）

- `terminate`: 1828（约 3.69 次/样本）
- `planning`: 536（约 1.08 次/样本）
- `python_execute`: 109
- `str_replace_editor`: 85
- `web_search`: 8

解释：该任务本质是“直接分类”，但流程中存在明显的编排与终止开销；在简单任务上，这类开销会显著稀释有效推理占比。

## 3. 混淆矩阵与错分模式

```json
{
  "dovish": {
    "dovish": 68,
    "neutral": 56,
    "hawkish": 5
  },
  "neutral": {
    "dovish": 86,
    "neutral": 151,
    "hawkish": 8
  },
  "hawkish": {
    "dovish": 57,
    "hawkish": 28,
    "neutral": 37
  }
}
```

关键观察：

- `hawkish` 识别能力最弱（122 条中仅 28 条识别正确），大量误判到 `dovish`/`neutral`。
- `neutral` 被误判到 `dovish` 的数量较高（86），说明模型对“风险、增长放缓、通胀描述”等语句存在偏鸽化倾向。

## 4. 错因分层（科研归因框架）

### A. 语义判别层（主要误差源）

- 现象：核心错误集中在 `reasoning_or_prompt_misalignment`。
- 机制：标签定义虽然明确，但模型在边界样本上对语义线索（风险提示、历史描述、前瞻措辞）映射不稳定。
- 影响：直接造成准确率损失。

### B. 流程控制层（次级但显著）

- 现象：`terminate` 调用频繁、少量循环、存在工具异常。
- 机制：多智能体编排对简单分类任务引入额外控制路径，增加不必要决策分叉。
- 影响：时延上升，并放大偶发性错误。

### C. 任务-架构匹配层（结构性因素）

- 现象：简单三分类任务使用规划+执行器的完整工作流。
- 机制：编排成本大于分工收益，导致“复杂流程处理简单任务”的效率损失。
- 影响：在准确率未显著提升的情况下，成本明显增加。

## 5. 关键案例类型（基于 failure_cases）

1. **语义边界误判类**  
   典型表现：输出给出详细解释，但最终标签与 gold 不一致；多见于 `neutral ↔ dovish`、`hawkish ↔ neutral`。

2. **流程冗余类**  
   典型表现：样本本应直接分类，却经历多次 `terminate` / 规划-执行往返。

3. **工具干扰类**  
   典型表现：少量样本出现无必要工具调用或工具异常，导致输出偏移。

## 6. 本轮错误分析结论

- 本轮错误的**主因是语义判别偏差**（而非单一工具问题）。
- 多智能体流程在该任务上体现出**显著的时延与流程开销**。
- 当前结果更支持“任务与架构匹配不足”的解释，而非“多智能体架构在所有任务上无效”。

## 7. 对后续实验的建议（不改代码前提）

- 进行严格消融：  
  `single-agent` vs `multi-agent(no planning)` vs `multi-agent(with planning)`。
- 补充统计指标：  
  Macro-F1、每类召回率、每样本工具调用数、每样本token消耗。
- 做多次重复实验：  
  报告均值、标准差、95% 置信区间，降低单次运行随机性影响。
