# 多智能体 Base 测试：科研评估报告（中文）

## 1. 研究目标与问题定义

本次 base 测试的核心目标是：在 FinBen 测试集上评估当前多智能体工作流（PlanningFlow + PlanningAgent + 执行器）作为“后续金融 Agent 构建基线”的有效性。

研究问题：

1. 在简单金融文本分类任务上，多智能体流程是否优于（或至少不劣于）基础模型直推理？  
2. 性能变化来自“模型语义能力差异”，还是“工作流编排开销与控制误差”？  
3. 该基线是否能为后续复杂金融任务（多步骤分析、工具协作）提供可复用参考？

## 2. 实验设置（可复现）

- 数据集：`Dataset/Finben/finben_test.json`
- 测试范围：全量（496）
- 超时设置：600 秒/样本
- 架构设置：Planning 启用（`PlanningFlow + executors(manus, finance)`）
- 输出工件：`predictions.jsonl`、`failure_cases.jsonl`、`summary.json`、`logs/`

## 3. 核心结果

- Accuracy：0.4980（247/496）
- 平均时延：34.308 秒/样本
- 工具调用总量：`planning=536`、`terminate=1828`、`python_execute=109`

解释：在该任务规模与难度下，流程编排成本显著，未表现出明确准确率收益。

## 4. 严谨归因：架构问题 vs 任务-架构匹配问题

### 4.1 不能简单得出“多智能体架构无效”

本次任务是短文本三分类，天然接近单步决策；多智能体优势（分工、跨步骤工具协作）难以发挥。  
因此，结果不能外推为“多智能体在所有金融任务上无价值”。

### 4.2 本轮更强证据支持“任务-架构匹配不足”

- 简单任务却引入规划、调度、终止控制链路；
- 过程开销明显增加（高时延、高 terminate 调用）；
- 准确率未体现相应收益。

结论：当前负效应主要体现为“复杂工作流处理简单任务”的匹配问题，其次才是局部实现细节问题。

## 5. 误差来源的科研分解

1. **语义判别误差（主因）**  
   `reasoning_or_prompt_misalignment` 数量最高，说明标签边界学习/执行不稳定。

2. **流程控制误差（次因）**  
   工具异常、循环迹象、过多终止行为会增加无效token与决策漂移。

3. **评测构念风险（方法学）**  
   需持续审查标签抽取与错误标签定义，避免“统计标签”与“真实错误”偏离。

## 6. 内部效度、外部效度与构念效度

### 6.1 内部效度（Internal Validity）

- 优点：全链路日志、工具参数、预测结果均已留痕，可审计。  
- 风险：在线模型与网络时延引入随机波动。  
- 建议：多次重复实验并报告均值/方差/置信区间。

### 6.2 外部效度（External Validity）

- FinBen 以货币政策语句分类为主；  
- 结果对“财报问答、估值推理、多文档综合分析”任务的可迁移性有限，需单独验证。

### 6.3 构念效度（Construct Validity）

- 仅看 Accuracy 不足以评价多智能体系统；  
- 应补充过程指标：工具效率、循环率、无效调用率、每样本 token 成本。

## 7. 对本次 Base 测试价值的评价

正向价值：

- 建立了可复现的“多智能体初始基线”；  
- 获得了结构化错误画像（语义误差、流程误差、工具误差）；  
- 为后续架构优化提供定量对照基准。

边界与局限：

- 当前任务过于简单，难体现多智能体分工优势；  
- 不宜将本次结论直接推广到复杂金融任务。

## 8. 后续实验建议（科研路线）

1. **做消融实验**  
   - 单智能体直推理  
   - 多智能体但关闭 planning  
   - 多智能体完整流程  
   对比 Accuracy、Macro-F1、时延、工具调用成本。

2. **做重复实验**  
   同设置重复 3~5 次，报告均值/标准差/95% CI。

3. **任务分层评估**  
   按任务复杂度分层（单句分类 / 多字段抽取 / 多步计算）评估多智能体边际收益。

4. **报告中明确结论边界**  
   强调“本次结果主要反映 simple classification 场景下的匹配问题”，避免过度外推。
