## 五、结论与后续建议

### 5.1 总体结论

1. **正确率**：58% → 67%，+9%，净增 18 个正确样本。
2. **耗时**：平均 -48%，P95 -64%，在更严格超时下仍大幅改善。
3. **稳定性**：timeout 24→1，rate_limit 8→0，tool_execution_error 5→0。
4. **流程效率**：zero-tool 空转 -71%，LLM 调用 -26%，python_execute -42%。

### 5.2 待改进点

1. **easy-test-180**：唯一新增 timeout，需单独分析流程与耗时瓶颈。
2. **14 个 Regressed 样本**：多为 same_args_repeated，需平衡「提前终止」与「保证正确性」，避免过早 terminate。
3. **52 个持续失败样本**：仍以 numeric_mismatch 为主，需加强推理与数值提取能力。

### 5.3 后续建议

- 对 easy-test-180 做单样本复现与 trace 分析。
- 对 regressed 样本做分层抽样，检查 anti-loop 与 terminate 策略是否过于激进。
- 对 52 个 still_wrong 做错因分类（公式理解、单位、精度等），针对性优化 prompt 或工具链。
