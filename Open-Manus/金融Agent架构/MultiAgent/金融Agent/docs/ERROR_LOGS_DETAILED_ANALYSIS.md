## 二、Planning 阶段明显问题的样本

### 2.1 easy-test-33 【Planning 严重错误：Agent 分配错误】

**题目**：请计算1998年4月和1998年11月税收减免的总投入，并将其与1999年11月的中小企业投入进行比较，求出差值；结果保留两位小数。  
**Gold**：-3.2

**Planning 输出**：
```
steps: [
  "[finance] 提取1998年4月税收减免金额和1998年11月税收减免金额",
  "[finance] 提取1999年11月中小企业投入金额",
  "[finance] 计算1998年4月和1998年11月税收减免的总投入",
  "[finance] 比较总投入与1999年11月中小企业投入，求出差值",
  "[finance] 输出结果并保留两位小数"
]
```

**问题**：
- 题目有图表数据，需从图中提取 1998年4月、11月、1999年11月 三组数值
- Planning 把 **所有提取步骤都分配给了 [finance]**，没有 [multimodal]
- Step 0 执行时：`base64_images_count=1` 但 `executor=Finance`，**Finance 不接收 base64_image**
- 结果：Finance 看不到图，只能凭空编造 50000、75000、120000
- Gold -3.2 说明正确单位可能是亿元/百万，数值完全不同

**根因**：**Planning 未识别题目含图表，错误地将提取任务分配给 Finance 而非 Multimodal**。

### 2.2 easy-test-22 【Planning 编造数值】

**题目**：What is the rate of return in percentage on the acquisition based on revenue generated by Indaba Group in the nine months ended March 31, 2016 over the total purchase price if we consider Goodwill as an earning asset?  
**Gold**：202.73

**Planning 输出**：
```
steps: [
  "[multimodal] Extract revenue generated by Indaba Group in nine months ended March 31, 2016 from provided images",
  "[finance] Compute rate of return = (Extracted_Revenue / 2000000) * 100"
]
```

**问题**：
- Planning 在 step 1 中 **编造了分母 2000000**
- 题目要求 "total purchase price if we consider Goodwill as an earning asset"
- Gold 公式：acquisition_cost_goodwill=1128004, revenue=2286772 → (2286772/1128004)*100 ≈ 202.73
- Planning 未要求提取 purchase price / Goodwill，直接假设 2000000

**根因**：**Planning 未正确解析题目，编造了关键参数**。同时 Multimodal 直接输出 `revenue_2016_q3 = None`，未从图中提取。

---

### 2.3 easy-test-18 【Planning 遗漏求和步骤】

**题目**：What is the total amount of Corporate notes and bonds of 2010 Fair Value, and Net sales of 2011?  
**Gold**：10514（= 1133 + 9381）

**Planning 输出**：
```
steps: [
  "[multimodal] Extract total amount of Corporate notes and bonds fair value for 2010 from image 1",
  "[multimodal] Extract Net sales amount for 2011 from image 2"
]
```

**问题**：
- 题目要求 "total amount of A **and** B"，即 **两个值之和**
- Planning 只规划了提取两个值，**没有规划求和步骤**
- 执行完 step 0 得 0（错误），step 1 得 8846
- 没有 step 2 把两个结果相加，最终 synthesize 时可能只用了最后一个值 8846

**根因**：**Planning 遗漏了「求和」步骤**，未理解题目要求的是两值之和。

### 3.5 easy-test-180 【Multimodal 代码语法错误 + 不修复】

**Planning**：正确，公式和数值都对  
**Multimodal 代码**：
```python
total_other_accrued_2013 = 243.7
 total_other_accrued_2012 = 247.9   # 第二行开头有多余空格
print(...)
```

**执行结果**：`unexpected indent (<string>, line 2)`

**后续**：Multimodal 反复输出 **完全相同** 的错误代码，被 anti-loop 拦截，无法修复。循环约 20 轮后 timeout。

**根因**：① **python_execute 输入的代码含非法缩进**（换行后多余空格）；② **Multimodal 不根据错误信息修改代码**，持续重复相同调用。

## 四、Finance 阶段问题的样本

### 4.1 easy-test-11 【Finance 公式符号错误】

**Planning**：正确，写的是 `27264 - 501 - 1063 = 25700`  
**Finance 执行**：
```python
cash_paid_to_suppliers = cost_of_goods_sold + inventory_decrease - accounts_payable_increase
# 即 27264 + 501 - 1063 = 26702
```

**根因**：Finance 把「Decrease in Inventory」理解为加项，正确应为减项。**Planning 给对了，Finance 执行时改错了**。

---

### 4.2 easy-test-41 【Finance 变量搞反】

**Multimodal 提取**：`liability_as_of_january_1_2004 = 2239`, `liability_as_of_december_31_2004 = 665`  
**Finance 计算**：
```python
liability_as_of_december_31_2004 = 2239  # 实际应为 665
liability_as_of_january_1_2004 = 665     # 实际应为 2239
net_change = 2239 - 665 = 1574
```

**Gold**：-1574 = 665 - 2239

**根因**：Finance 把 1 月和 12 月的值 **对调**，导致符号相反。

## 五、python_execute / 工具层问题

### 5.1 easy-test-33 【元组输出未正确解析】

Finance step 0 代码：
```python
tax_reduction_april_1998, tax_reduction_november_1998  # 最后一行是表达式，输出元组
```

**执行输出**：`(50000, 75000)`

后续 step 2 计算 `total_tax_reduction = tax_reduction_april_1998 + tax_reduction_november_1998` 时，**跨 step 变量不共享**，Finance 在 step 2 的上下文中没有这两个变量，可能从历史输出解析或重新赋值。最终 125000 - 120000 = 5000，而 gold 是 -3.2，说明提取的数值本身错误（单位/来源）。

**根因**：① Planning 把提取任务错误分配给 Finance（见 2.1）；② 若需跨 step 传递，当前流程可能未正确传递变量。

## 七、Planning 阶段问题归纳

| 问题类型 | 样本 | 表现 |
|----------|------|------|
| **Agent 分配错误** | easy-test-33 | 有图却全部分配给 Finance，应用 Multimodal |
| **编造关键参数** | easy-test-22 | 分母 2000000 为编造，未从题目/图提取 |
| **遗漏步骤** | easy-test-18 | 只规划提取，未规划求和 |

**建议**：
1. Planning 需明确：若题目涉及图表数据，提取步骤必须分配 [multimodal]
2. Planning 不得在步骤中写死具体数值，应从题目/图中提取
3. Planning 需完整理解题目要求（如「A 与 B 之和」需包含求和步骤）
