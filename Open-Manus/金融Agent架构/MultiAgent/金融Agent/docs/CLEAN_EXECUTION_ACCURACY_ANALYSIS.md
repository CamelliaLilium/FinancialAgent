# 正常执行正确率分析

## 目的

区分**架构/流程异常**与**智能体/工具能力**，统计在「正常完成执行」的样本中，智能体+工具能否解决任务。

## 流程异常定义（排除后不反映能力）

| 类型 | 说明 |
|------|------|
| `timeout` | 超时未完成 |
| `runtime_error` | 运行时崩溃 |
| `empty_output` | 无输出 |
| `rate_limit` | API 限流影响执行 |
| `repeated_tool_loop` | 重复调用触发最大步数强制终止 |
| `tool_execution_error` | 工具执行失败 |
| `multimodal_input_missing` | 图像路径未解析 |
| `multimodal_vision_failure` | 视觉提取失败 |
| `same_args_repeated` | 同一工具同一参数重复调用（冗余） |
| `zero_tool_rounds >= 5` | Multimodal 多轮未调工具空转 |

## 200 样本统计结果

### 1. 最严格：排除所有流程异常

- **正常执行**：19
- **正确**：11
- **正确率**：**57.9%** (11/19)

### 2. 放宽 same_args_repeated：仅排除严重流程异常

（保留 same_args_repeated，因其多为冗余调用，不一定导致错误）

- **正常执行**：153
- **正确**：102
- **正确率**：**66.7%** (102/153)

### 3. 仅 same_args_repeated 的样本（无 repeated_tool_loop、zero_tool<5）

- **样本数**：139
- **正确**：94
- **正确率**：**67.6%** (94/139)

## 被排除的流程异常分布

| 原因 | 样本数 |
|------|--------|
| same_args_repeated | 147 |
| timeout | 24 |
| zero_tool_rounds >= 5 | 18 |
| tool_execution_error | 5 |
| rate_limit | 3 |

## 结论

1. **能力上限**：在排除严重流程异常（超时、重复循环、0-tool 空转、工具错误、限流）后，**正常执行正确率约 66.7%**，可视为当前架构下智能体+工具的能力上限。

2. **same_args_repeated**：147 个样本存在，但多数仍能给出正确答案；更多是效率问题，而非能力问题。

3. **主要瓶颈**：
   - **timeout (24)**：执行时间过长
   - **zero_tool_rounds >= 5 (18)**：Multimodal 未调工具空转
   - **tool_execution_error (5)**：代码/工具执行失败

4. **与原始指标对比**：
   - 原始 acc：58% (116/200)
   - 流程完成 acc：65.9% (116/176)
   - 正常执行 acc：**66.7%** (102/153)

说明：在「正常执行」样本中，正确率与「流程完成」样本接近，架构与工具已具备一定任务解决能力，主要问题集中在流程稳定性与执行效率。
